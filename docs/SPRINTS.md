# Planejamento das Sprints

## Sprint 1 - Fundacao

- Estrutura monorepo criada.
- Entidades principais definidas.
- Contrato inicial da API documentado.

## Sprint 2 - Autenticacao

- Login JWT.
- Roles `ALUNO`, `PROFESSOR` e `ADMIN`.
- Rotas protegidas no backend.
- Sessao salva no web e mobile.

## Sprint 3 - QR Code e Presenca

- Professor cria aula pelo web.
- Backend gera `qrToken`.
- Web mostra QR Code.
- Mobile escaneia QR Code.
- Backend registra presenca.

## Sprint 4 - Dashboards

- Relatorio por turma.
- Percentual de frequencia por aluno.
- Historico de presencas no app do aluno.

## Sprint 5 - Polimento

- Testar fluxo completo com os usuarios de demo.
- Melhorar mensagens de erro.
- Validar uso em celular real com Expo Go.

## Sprint 6 - TCC

- Finalizar README.
- Inserir diagramas no relatorio.
- Preparar prints das telas.
- Criar slides com problema, solucao, arquitetura, demo e conclusao.
